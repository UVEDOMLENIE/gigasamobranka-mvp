## Слайд 3 — Проблема и пользователи

**Проблема:**
Учитель тратит 2–3 часа на ручное составление карточек, тестов, планов урока. Рутина отнимает время от живого общения с детьми.

**Целевая группа:**
Учителя начальной и средней школы (1–11 класс), методисты, репетиторы.

**Сценарий столкновения:**
1. Учитель готовится к уроку — нужно 20–30 карточек по теме.
2. Открывает Word → копирует текст → формулирует вопросы → форматирует → печатает.
3. 2 часа ушло. На разговор с учениками — 0 минут.

---

## Слайд 4 — Решение

**Идея:** Загрузить учебник (PDF/DOCX/TXT) → LLM генерирует карточки → учитель правит → ссылка ученикам → результаты → печать.

**Пользовательские сценарии:**
1. Учитель загружает материал → получает набор за 30 сек.
2. Правит карточки inline → сохраняет.
3. Даёт ссылку ученикам → ученик проходит на телефоне.
4. Смотрит результаты → печатает DOCX.

**Сравнение с существующими:**

| | Quizlet | Anki | ГигаСамобранка |
|---|---|---|---|
| Генерация из PDF/DOCX | ❌ | ❌ | ✅ |
| Редактирование | ❌ | ⚠️ | ✅ |
| Режим ученика + статистика | ❌ | ❌ | ✅ |
| Печать A4 / DOCX | ❌ | ❌ | ✅ |
| LLM-генерация на русском | ❌ | ❌ | ✅ |

## Слайд 5 — Техническая реализация

**Архитектура:**

```
[Браузер] ←→ [Next.js 16 App Router]
                │
                ├── API Routes (Server Actions)
                │   ├── /api/generate → LLM (Scarlex / GigaChat)
                │   ├── /api/upload → Парсинг (mammoth/pdf-parse)
                │   ├── /api/sets/[id] → CRUD карточек
                │   ├── /api/sets/[id]/docx → DOCX экспорт
                │   └── /api/sessions → Статистика учеников
                │
                ├── SQLite (libsql) → Drizzle ORM
                │   ├── sets, cards, sessions, answers
                │   └── Vercel Blob backup/restore (prod persistence)
                │
                └── LLM Providers
                    ├── Scarlex (Claude Sonnet 4.6) — основной
                    ├── GigaChat — резервный
                    └── Mock (regex) — fallback без ключа
```

**Стек:** Next.js 16, React 19, Tailwind v4, TypeScript, Drizzle ORM, libsql, KaTeX.

Схема: `docs/defence/architecture.mmd` (Mermaid)
