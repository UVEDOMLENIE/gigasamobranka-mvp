## Проблемы

### 1. Scarlex API нестабилен (главный затык)

Прогнал руками `curl`:

| Модель | Статус | Комментарий |
|--------|--------|-------------|
| `claude-opus-4-7` | 429 | Rate limit. Полностью забит |
| `claude-sonnet-4-7` | 200 + `"Service temporarily unavailable"` | Сервис лежит |
| `gpt-4o` | 402 Quota exceeded | Квота на ключ исчерпана |
| `claude-haiku-4-7` | 200 + нормальный ответ | **Единственная рабочая** |

Один тестовый запрос `"Hi"` через Haiku прошёл. Несколько подряд → опять 429.

**Вывод**: Scarlex имеет очень жёсткий rate-limit на ключ (предположительно 1–2 запроса в минуту на модель). Использовать его «в боевом режиме» без retry / задержек нельзя.

### 2. SSE-формат вместо JSON

`stream: false` в запросе **игнорируется** Scarlex'ом — он всегда возвращает `chat.completion.chunk` через SSE (`data: {...}\n`). Это починено в `lib/llm/gigachat.ts` (склейка `delta.content` из всех чанков).

### 3. Mock выдаёт markdown-мусор

Если LLM падает → fallback на regex-mock. Он выдирает из README:
- `[Глава 1](#глава-1)` (markdown-ссылки)
- `«перезагрузи голову»` (любые слова с заглавной)
- `# Дефекты` (заголовки целиком)

Из-за этого пользователь думает «дичь генерирует нейронка». На самом деле нейронка вообще не вызывалась.

**Решение для презентации**: не использовать README.md как материал. Использовать готовые демо-карточки (`/api/demo?id=winter-vocab`) — они работают идеально.
