# Sprint 4 — Качество генерации карточек

Дата: 2026-05-28.
Исполнитель: Devin (Claude Sonnet, Cognition AI).
Инициатор: UVEDOMLENIE — "генерация карточек через жопу работает".

## Контекст

Пользователь задеплоил MVP на Vercel (`gs-mvp-six.vercel.app`), но генерация карточек выдавала шаблонные заглушки вместо осмысленных вопросов. Жюри хакатона СберОбразование × Школа 21 должно увидеть рабочий продукт.

## Диагностика

### Найденные проблемы

1. **Mock вместо нейронки** — дефолтный `LLM_PROVIDER=mock` в `.env.example` означал, что без явной настройки env на Vercel генерация шла через regex-парсер (`lib/llm/mock.ts`), а не через Scarlex/Claude.

2. **Слабая модель по умолчанию** — даже при включённом Scarlex дефолтная модель была `claude-haiku-4-7` (быстрая, но простая), а не `claude-sonnet-4.6` (рекомендуемая).

3. **Бедный промпт** — системный промпт был однострочным ("Ты методист. Ответь JSON-массивом..."), без методических инструкций по качеству карточек.

4. **Обрезка контекста** — исходный текст обрезался до 2000 символов на файл, теряя содержимое больших документов.

5. **Невалидный ID модели в `.env.example`** — указан `claude-opus-4-7`, которого нет в `SCARLEX_MODELS.md`. Валидные: `claude-sonnet-4.6`, `claude-sonnet-4`, `claude-opus-4.7`, `claude-opus-4.6`, `claude-haiku-4-7`.

### Как нашли

- Прочитали `lib/llm/gigachat.ts` — логика выбора провайдера, fallback-цепочка
- Прочитали `lib/llm/mock.ts` — regex-парсер, шаблонные заглушки
- Прочитали `app/settings/page.tsx` — UI настроек, дефолтные значения
- Прочитали `.env.example` — шаблон переменных окружения
- Прочитали `docs/sprints/02-stability/SCARLEX_MODELS.md` — список валидных моделей

## Что сделали

### Изменённые файлы

| Файл | Что изменено |
|------|-------------|
| `lib/llm/gigachat.ts` | Новый SYSTEM_PROMPT с методическими правилами; дефолтная модель → sonnet-4.6; промпт buildPrompt переструктурирован; лимит контекста 2000→4000 |
| `app/settings/page.tsx` | Дефолт модели → sonnet-4.6; порядок моделей в dropdown (sonnet первый); обновлён help-текст |
| `.env.example` | LLM_PROVIDER=scarlex (не mock); SCARLEX_MODEL=claude-sonnet-4.6; USE_MOCK_LLM=false |

### Детали изменений

**1. Системный промпт** (новый `SYSTEM_PROMPT` в `gigachat.ts`):
```
Ты опытный методист-составитель учебных карточек.

ПРАВИЛА:
- Вопрос должен быть конкретным, проверяющим понимание материала
- Ответ — краткий, точный, информативный (1–3 предложения)
- Не повторяй вопрос в ответе
- Используй терминологию из исходного текста
- difficulty: easy (определения, факты), medium (применение, связи), hard (анализ, синтез)
- source — имя файла-источника

Формат ответа — ТОЛЬКО JSON-массив, без пояснений.
```

**2. Пользовательский промпт** (переструктурирован `buildPrompt`):
```
Сгенерируй ровно {count} учебных карточек.
Предмет: {subject}. Класс: {grade}. Тема: «{topic}». Сложность: {difficulty}.
Исходный материал:
{контекст до 4000 символов на файл}
```

**3. Модель по умолчанию**: `claude-haiku-4-7` → `claude-sonnet-4.6` (баланс качества и скорости).

**4. UI настроек**: sonnet-4.6 первый в списке с пометкой "(рекомендуем — default)".
