## Проблемы с продом (НЕ исправлены, идут в S03)

### 1. SQLite в `/tmp` НЕ персистентен между инстансами

**Симптом**: пользователь создаёт набор → через несколько минут (когда serverless instance умирает) → набор исчезает.

**Причина**: Vercel serverless functions имеют read-only filesystem кроме `/tmp`. Но `/tmp` уникален для каждого инстанса и сбрасывается. Это не БД, это эфемерный кэш.

**Что было в `docs/sprints/02-stability/TASKS.md` под T4**:
> ВНИМАНИЕ: SQLite на Vercel работает только в read-only (serverless). Для prod нужно либо Turso (libsql cloud), либо переключить БД. Это **отдельная задача T6**.

Кими прочитал, но **не послушался** — пошёл по самому быстрому пути. Smoke прошёл потому что smoke создаёт и читает набор в одном запросе (одна serverless-инстанс жив).

**Решение**: миграция на **Turso** (libsql cloud, бесплатно 5GB, 1 минута регистрация через GitHub).
