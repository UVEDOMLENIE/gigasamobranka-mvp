# Требования организаторов (сжатая выжимка)

Источник: вебинар + обратка 24.05.2026 + финальное ТЗ.

## Главная формула

```
Учитель загружает учебные материалы
→ сервис извлекает текст
→ LLM генерирует карточки
→ учитель редактирует
→ запускает плеер на интерактивной доске
→ даёт ссылку ученикам
→ ученики проходят индивидуально
→ учитель видит результаты
→ опционально печатает
```

## Обязательные требования

1. **Карточки** — основной формат. Не распыляться на викторины/контрольные в MVP.
2. **Ссылка ученикам** — общая ссылка на набор, без регистрации, по имени/нику.
3. **Результаты учителю** — таблица прохождений по набору.
4. **KaTeX / LaTeX** — формулы должны рендериться.
5. **Печать** — A4, готовый к выводу на принтер.
6. **Детский UX** — крупный шрифт, прогресс, похвала, простые кнопки.
7. **Источники цифр** — если указываем расход/экономику, давать ссылку на тариф.
8. **Учитель — не IT-шник** — один экран, одна задача, простота.

## Что орги похвалили

- Идея сильная.
- Генерация из учебных материалов, а не «из воздуха».
- Рекомендовали ограничиться карточками для прототипа.

## Что орги спросили / уточнили

- Ссылка для класса или индивидуально? → Индивидуально, с результатами.
- Как учитель видит результаты? → `/sets/[id]/results`.
- Считали ли себестоимость? → Табличка в презентации (~200–300 ₽/учитель/мес).
- Выходной файл без доработки? → A4-сетка, шрифт ≥14pt.

## Для презентации/защиты

- Упомянуть roadmap: викторины, раздатки, mindmap — но не как часть MVP.
- Показать экономику с открытыми источниками (GigaChat tariff, Vercel free tier).
- Продемонстрировать демо-сценарий глазами учителя.
